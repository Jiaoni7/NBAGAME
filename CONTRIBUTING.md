# 贡献说明

感谢参与这个项目。

当前仓库优先服务小团队协作，同时保持对外可读、可运行、可审阅。外部贡献同样欢迎，但请尽量保持改动聚焦，并避免混入团队私有材料。

## 开始前

- 阅读 [README.md](README.md)
- 阅读 [DEVELOPMENT.md](DEVELOPMENT.md)
- 如需了解当前结构与文档入口，阅读 [docs/README.zh-CN.md](docs/README.zh-CN.md)
- 尽量让一次改动只服务一个主要目的

## 分支命名

使用以下前缀之一：

- `feat/<name>`
- `fix/<name>`
- `docs/<name>`
- `chore/<name>`

## 提交信息

使用简短前缀：

- `feat:`
- `fix:`
- `docs:`
- `chore:`
- `refactor:`

## Pull Request 要求

- 说明改了什么，以及为什么改
- 写明做过哪些验证
- 不要把无关清理混进同一个 PR
- 如果命令、结构、流程或公开文档入口变化了，同步更新相关文档

## 不要提交的内容

- `skills/`、`skills.md` 或项目特化的本地 `SKILL.md`
- `.codex/`、`.claude/`
- 私有提示词、调研笔记、内部审计草稿
- 本地日志、缓存、临时文件
- 机器专属脚本、路径或环境文件

## 审阅重点

审阅时优先关注：

- 正确性
- 回归风险
- 仓库卫生
- 文档是否仍然准确
- 是否意外暴露本地私有资产
