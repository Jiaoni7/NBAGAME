# 王朝传奇 · 迭代 1.1 更新文档

> 发布日期：2026-06-03
> 主题：**联盟模拟系统**（单人模式）→ **东西部 30 队重构**（见第八节）
> 涉及文件：`game.js`、`index.html`、`sw.js`、`manifest.webmanifest`

---

## 一、本次更新概览

为单人模式（`gameMode === 'single'`）新增一整套 **联盟模拟系统**，让玩家在打比赛的同时，整个 8 队联盟同步运转：球队排名、个人/球队数据榜、季后赛对阵图、常规赛与总决赛颁奖、赛季间选秀大会全部联动。

核心设计约束：**球队战力真实影响胜率**——AI 球队强度只跟赛季数挂钩，不会随玩家战力水涨船高，因此堆阵容、签新秀能切实提升战绩与排名。

---

## 二、功能清单

### 1. 联盟模拟系统（8 队）
- 联盟固定 **8 支球队**：玩家 1 队 + AI 7 队。
- AI 球队优先采用 **2025-26 赛季真实首发名单**（详见功能 7），每队 5 名真实球员。
- 玩家每打一场真实比赛，AI 各队按 `str /(str + oppStr)` 概率同步模拟一轮，常规赛战绩并行推进。
- AI 强度锚定赛季：`aiStrengthBase = 150 + (赛季-1) × 62`，与玩家战力解耦。
- 常规赛固定 **12 场**（`REGULAR_GAMES`）。

### 2. 数据中心 UI（球队面板内入口）
入口标签：📊 排名 / 📈 数据榜 / 🏆 对阵图 / 🏅 奖项 / 📜 王朝史
- **球队排名**：按胜率排序，前 8 名进入季后赛并标注种子，显示胜/负/胜率/战力，"我"高亮。
- **数据榜**：
  - 个人榜：基础数据（得分/篮板/助攻/抢断/盖帽王）与高阶数据（PER / TS% / USG% / EFF）各取前 5。
  - 球队榜：球队进攻榜（场均得分总和）。
- 数据采用 **种子化随机**（FNV-1a → mulberry32），多次渲染结果稳定。

### 3. 季后赛对阵图
- 常规赛结束后按战绩种子排定对阵（1v8 / 4v5 / 2v7 / 3v6）。
- **7 局 4 胜制**（`SERIES_WIN = 4`），8 → 4 → 2 → 1 可视化呈现。
- 玩家系列赛与 AI 系列赛并行推进，玩家所在对阵高亮"进行中"，最终展示总冠军。

### 4. 奖项系统
- **常规赛颁奖典礼**：MVP / DPOY / ROY / MIP / 6MOY，附获奖者数据。
- **总决赛 FMVP**：夺冠后单独颁奖弹窗。
- 历史奖项沉淀至 **王朝史** 标签。

### 5. 选秀大会（赛季之间）
- 赛季结束进入 `offseason` 阶段，举办选秀大会。
- 提供数名新秀候选，玩家选 1 名加入球队。
- 新秀作为 **替补深度** 提升球队战力（`rating × 0.4`），最多保留 4 名；下赛季有资格竞争最佳新秀（ROY）。

### 6. 存档/读档同步 + 资源更新
- 存档新增字段：`rookies`、`league`、`awards`、`fmvp`、`history`、`pendingDraft`。
- 读档对上述字段做兼容默认值还原。
- 修复对阵图存读档后对象引用失效问题：胜者比对改用 `.name` 而非 `===`。

### 7. 真实名单与 5 人首发阵容（1.1 修订）
- **新增 2025-26 赛季真实首发名单库 `REAL_ROSTERS`**：覆盖全部 16 支 AI 球队，每队 5 名真实首发（PG/SG/SF/PF/C），含本赛季重大变动：
  - 湖人：东契奇 + 詹姆斯 + 里夫斯 + 八村垒 + 艾顿
  - 马刺：文班亚马 + 福克斯 + 卡斯尔
  - 独行侠：戴维斯 + 状元弗拉格 + 汤普森
  - 太阳：布克 + 杰伦·格林（KD 已离队）
  - 雄鹿：字母哥 + 特纳（利拉德已离队）等
- **`buildAiTeam`**：优先采用真实名单，球员评级随赛季缓慢成长（`grow = (赛季-1) × 1.2`），保持后期挑战性；球队战力仍以"赛季"为锚 + 真实名单整体强度微调，**不随玩家战力膨胀**（保留原设计约束）。
- **修复对手阵容只显示 3 人的问题**：`genOpponent` 由 `slice(0,3)` 改为完整 **5 人首发**，按 PG→C 位置顺序展示；非联盟兜底生成器 `genOppLineup` 同步改为生成 5 人。
- **对手预览**：每名球员前显示位置标签（PG/SG/SF/PF/C）。
- **旧存档兼容**：检测到 `nextOpp` 阵容不足 5 人时，自动按新规则重建。

## 三、赛季状态机

```
regular（常规赛 12 场）
   │  打满 12 场 → 计算奖项、排定种子
   ├─ 进入前 8 ─→ playoff（季后赛 7 局 4 胜）
   │                 │ 系列赛胜 → 晋级下一轮 / 夺冠 → FMVP
   │                 │ 系列赛负 → 模拟剩余对阵 → offseason
   └─ 未进前 8 ─→ 颁奖典礼 → offseason
offseason（选秀大会）→ 选秀完成 → 下一赛季 regular
```

## 四、技术要点

| 项 | 说明 |
|---|---|
| 模块模式 | 原生 JS IIFE，能力通过 `App.` 对象暴露，HTML 用 `onclick="App.fn()"` 调用 |
| 随机算法 | 种子化 PRNG（FNV-1a 哈希 → mulberry32），保证数据稳定可复现 |

| 高阶数据 | PER、TS%、USG%、EFF |
| AI 模拟 | 概率胜负 `str /(str + oppStr)`，强度仅与赛季数相关 |
| 弹层 | 复用 `ensureModal(id)` + `.modal-mask` / `.card` 结构 |

## 八、东西部 30 队联盟重构（1.1 二次修订）

> 本节为对前文「联盟模拟系统」的重大重构，**覆盖**第一/三节中关于「8 队」「12 场」「8 队季后赛」的描述。

### 1. 30 队 + 东西部分区
- 联盟扩展为 **30 支球队**（`LEAGUE_SIZE = 30`）：玩家固定 **东部**（`conf:'E'`）+ 14 支东部 AI + 15 支西部 AI。
- 新增分区常量与工具：`EAST_TEAMS[15]`、`WEST_TEAMS[15]`、`confOf(name)`；`createTeam`/`buildAiTeam`/`playerLeagueTeam` 均携带 `conf` 字段。
- `REAL_ROSTERS` 补齐至全部 30 队（新增猛龙、骑士、活塞、步行者、老鹰、黄蜂、魔术、奇才、雷霆、开拓者、爵士、国王、火箭、灰熊等），含真实交易（东契奇→湖人、文班亚马→马刺、KD→火箭、杰伦·格林→太阳、拉文→国王、状元弗拉格→独行侠）。
- `buildLeague` 重写为分区构建；`ensureLeague` 在 `teams.length < LEAGUE_SIZE` 时自动重建旧联盟。

### 2. 分区排名
- `standings(team, conf)` 支持按分区过滤；新增 `playerConfRank(team)`。
- 排名页新增 **东部 / 西部** 子页签（`App.setStandingsConf`），状态变量 `standingsConf`。
- 种子区可视化：**1–6 直接晋级**（蓝色边条 `z-direct`）、**7–10 附加赛**（绿色边条 `z-playin` + `.seed.playin`）、**>10 无缘**（`.seed.out`）。

### 3. 常规赛 82 场
- `REGULAR_GAMES` 由 12 改为 **82**，AI 各队并行推进至 82 场后统一计算分区种子。

### 4. 附加赛（Play-In Tournament）
- 触发条件：常规赛结束后分区排名 **7–10 名**进入附加赛。
- 赛制：Game A（7v8，胜者得第 7 种子）、Game B（9v10）、Game C（A 负者 vs B 胜者，胜者得第 8 种子）。
- 新增模块：`byId`、`simGame`、`simPlayinConf`、`startPlayin`、`finishPlayinQualify`、`eliminateInPlayin`；玩家最多打 2 场单场决胜，其余由 `simGame` 模拟。

### 5. 季后赛四轮（东西部双区对阵）
- 对阵数据模型：`{east:{rounds:[r1,r2,r3]}, west:{rounds:[r1,r2,r3]}, finals, champion}`。
- 四轮命名（`PLAYOFF_ROUND_NAMES`）：**首轮 → 分区半决赛（次轮）→ 分区决赛 → 总决赛**。
- 两分区通过 `simConfRound` **同步推进**，玩家仅打本分区，直到总决赛才与对面分区冠军相遇。
- 新增/重写：`pairOf`、`confTop8`、`buildBracket`、`findPlayerMatchup`（finals→east→west）、`playerSeriesOpponent`、`currentPlayoffRound`、`simSeries`、`simConfRound`、`advanceBracket`、`simulateRemainingBracket`。
- 对阵图渲染（`renderBracket`）分东部 / 总决赛 / 西部三块，新增 `.bk-conf`、`.bk-conf-h`（`.east`/`.west`/`.gold`）、`.bk-conf.finals` 样式。

### 6. 赛季状态机（更新版）

```
regular（常规赛 82 场）
   │  打满 82 场 → 计算奖项、排定分区种子
   ├─ 分区 1–6 名 ─→ playoff（首轮 → 分区半决赛 → 分区决赛 → 总决赛，均 7 局 4 胜）
   ├─ 分区 7–10 名 ─→ playin（附加赛 A/B/C）→ 晋级则进 playoff，否则 offseason
   └─ 分区 >10 名 ─→ 颁奖典礼 → offseason
offseason（选秀大会）→ 选秀完成 → 下一赛季 regular（重置 playin/playoff 状态）
```

### 7. 存读档与兼容
- 存档新增字段：`conf`、`playin`。
- **向后兼容**：检测到旧 8 队联盟或旧对阵图结构时，自动置空 `league` 并将 `playoff`/`playin` 阶段重置回 `regular`，下次进入自动按新规则重建 30 队联盟。

### 8. 文案与资源
- `renderRules` 文案更新为「30 队 / 东西部 / 82 场 / 附加赛 / 四轮季后赛」，移除已删除的 `PLAYOFF_NEED` 引用。
- Service Worker 缓存版本 `nba-dynasty-v8` → **`v11`**（v9/v10/v11 对应重构迭代）。
