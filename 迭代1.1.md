# 王朝传奇 · 迭代 1.1 更新文档

> 发布日期：2026-06-03
> 主题：**联盟模拟系统**（单人模式）
> 涉及文件：`game.js`、`index.html`、`sw.js`

---

## 一、本次更新概览

为单人模式（`gameMode === 'single'`）新增一整套 **联盟模拟系统**，让玩家在打比赛的同时，整个 8 队联盟同步运转：球队排名、个人/球队数据榜、季后赛对阵图、常规赛与总决赛颁奖、赛季间选秀大会全部联动。

核心设计约束：**球队战力真实影响胜率**——AI 球队强度只跟赛季数挂钩，不会随玩家战力水涨船高，因此堆阵容、签新秀能切实提升战绩与排名。

---

## 二、功能清单

### 1. 联盟模拟系统（8 队）
- 联盟固定 **8 支球队**：玩家 1 队 + AI 7 队。
- 每支球队自动生成球员阵容与数据（基础 + 高阶）。
- 玩家每打一场真实比赛，AI 各队按 `str /(str + oppStr)` 概率同步模拟一轮，常规赛战绩并行推进。
- AI 强度锚定赛季：`aiStrengthBase = 150 + (赛季-1) × 62`，与玩家战力解耦。
- 常规赛固定 **12 场**（`REGULAR_GAMES`）。

### 2. 数据中心 UI（球队面板内入口）
入口标签：📊 排名 / 📈 数据榜 / 🏆 对阵图 / 🏅 奖项 / 📜 王朝史
- **球队排名**：按胜率排序，前 8 名进入季后赛并标注种子，显示胜/负/胜率/战力，"我"高亮。
- **数据榜**：
  - 个人榜：基础数据（得分/篮板/助攻/抢断/盖帽王）与高阶数据（PER / TS% / USG% / EFF）各取前 5。
  - 球队榜：球队进攻榜（场均得分总和）。
- 数据采用 **种子化随机**（FNV-1a → mulberry32），多次渲染结果稳定。

### 3. 季后赛对阵图
- 常规赛结束后按战绩种子排定对阵（1v8 / 4v5 / 2v7 / 3v6）。
- **7 局 4 胜制**（`SERIES_WIN = 4`），8 → 4 → 2 → 1 可视化呈现。
- 玩家系列赛与 AI 系列赛并行推进，玩家所在对阵高亮"进行中"，最终展示总冠军。

### 4. 奖项系统
- **常规赛颁奖典礼**：MVP / DPOY / ROY / MIP / 6MOY，附获奖者数据。
- **总决赛 FMVP**：夺冠后单独颁奖弹窗。
- 历史奖项沉淀至 **王朝史** 标签。

### 5. 选秀大会（赛季之间）
- 赛季结束进入 `offseason` 阶段，举办选秀大会。
- 提供数名新秀候选，玩家选 1 名加入球队。
- 新秀作为 **替补深度** 提升球队战力（`rating × 0.4`），最多保留 4 名；下赛季有资格竞争最佳新秀（ROY）。

### 6. 存档/读档同步 + 资源更新
- 存档新增字段：`rookies`、`league`、`awards`、`fmvp`、`history`、`pendingDraft`。
- 读档对上述字段做兼容默认值还原。
- 修复对阵图存读档后对象引用失效问题：胜者比对改用 `.name` 而非 `===`。

## 三、赛季状态机

```
regular（常规赛 12 场）
   │  打满 12 场 → 计算奖项、排定种子
   ├─ 进入前 8 ─→ playoff（季后赛 7 局 4 胜）
   │                 │ 系列赛胜 → 晋级下一轮 / 夺冠 → FMVP
   │                 │ 系列赛负 → 模拟剩余对阵 → offseason
   └─ 未进前 8 ─→ 颁奖典礼 → offseason
offseason（选秀大会）→ 选秀完成 → 下一赛季 regular
```

## 四、技术要点

| 项 | 说明 |
|---|---|
| 模块模式 | 原生 JS IIFE，能力通过 `App.` 对象暴露，HTML 用 `onclick="App.fn()"` 调用 |
| 随机算法 | 种子化 PRNG（FNV-1a 哈希 → mulberry32），保证数据稳定可复现 |
| 球员数据 | 基于位置（PG/SG/SF/PF/C）的统计模板，按评级缩放为场均数据 |
| 高阶数据 | PER、TS%、USG%、EFF |
| AI 模拟 | 概率胜负 `str /(str + oppStr)`，强度仅与赛季数相关 |
| 弹层 | 复用 `ensureModal(id)` + `.modal-mask` / `.card` 结构 |
